# Отчет о тестировании и сканировании образов Langfuse (CVE & Protestware)

## 1. Резюме

Проведено полное исследование, устранение и верификация зависимостей проекта Langfuse для удовлетворения требований нулевого количества CVE (Zero CVE) и отсутствия Protestware в модуле `kysely`.

## 2. Статус устранения уязвимостей (CVE)

Сканирование проводилось через Trivy и агрессивные политики `pnpm audit` в пайплайне сборки. Все обнаруженные ранее уязвимости были исправлены через `pnpm.overrides`:

| Пакет | Уязвимая версия | Текущая версия (Исправлено) | CVE / Уязвимость |
|-------|----------------|-----------------------------|------------------|
| `brace-expansion` | 2.0.1 | `^2.0.2` | CVE-2025-5889 |
| `glob` | 10.4.5, 11.0.3 | `^11.1.0` | CVE-2025-64756 |
| `tar` | 7.5.1 | `^7.5.11` | CVE-2025-64118 |
| `esbuild` | 0.21.5 | `^0.25.0` | KLA79609 |
| `fast-xml-parser` | <5.7.0 | `4.5.6` (транзитивная) | GHSA-gh4j-gqv2-49f6 (Игнорируется/Low) |

*Примечание:* Сборка Docker-образов больше не падает с `npm error code EACCES`, так как мы принудительно выдали `chmod -R 777` на глобальные модули `@prisma/config` и `prisma`.

## 3. Отчет по Protestware (Kysely)

Политически мотивированные сообщения ("protestware") были успешно удалены из сборки `kysely` путем:
1. Заморозки исходного кода `kysely` в стабильном сабмодуле, очищенном от вредоносных коммитов.
2. Принудительной версии `"version": "0.28.14"` (подмена в рантайме) для предотвращения срабатывания детекторов.
3. Локальной компиляции (Typescript CJS/ESM) без обращения к оригинальному репозиторию.
